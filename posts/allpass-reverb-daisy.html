Building a Plate Reverb on the Daisy Seed with Allpass Diffusors

A walkthrough of the Schroeder reverb architecture, from the math behind allpass filters to real-time C++ implementation on bare-metal hardware. We cover the delay network, diffusion parameters, and the surprisingly tricky pre-delay smoothing.

Why reverb in hardware?

There's something satisfying about reverb that runs entirely in a microcontroller — no latency from USB audio, no driver issues, just audio in and audio out at the speed of the hardware. The Daisy Seed makes this genuinely approachable: it runs at 480 MHz with a hardware floating-point unit and 64 MB of SDRAM for delay buffers.

I wanted a plate-style reverb for a Eurorack module: lush, diffuse tail, Gilmour-friendly pre-delay up to around 80ms, and stereo spread from a mono input. The Schroeder architecture from the 1960s is still a solid starting point.

The Schroeder architecture

A Schroeder reverb consists of two stages:

  1. Parallel comb filters — create a dense reflection pattern
  2. Series allpass filters — diffuse the comb filter output to destroy the periodicity

The comb filters are feedback delay lines (FDLs) of different lengths. Each one rings at its own frequency, and the combination of four to six of them creates a pseudo-random reflection density. The problem is that they sound metallic — you can hear the individual pitches. That's where allpass filters come in.

Allpass filters as diffusors

An allpass filter has flat magnitude response but non-linear phase. In the context of reverb, it scatters the time domain without changing the frequency content. The transfer function is:

H(z) = (-g + z⁻ᴺ) / (1 - g·z⁻ᴺ)

Where N is the delay length in samples and g is the feedback gain (typically 0.5–0.7 for musical diffusion). The implementation in C++ on the Daisy is straightforward: 

class AllpassFilter {
  float* buf;
  int size, writeIdx;
  float g;

public:
  AllpassFilter(float* buffer, int delaySamples, float gain)
    : buf(buffer), size(delaySamples), writeIdx(0), g(gain) {
    memset(buf, 0, sizeof(float) * size);
  }

  float process(float in) {
    float delayed = buf[writeIdx];
    float output  = -g * in + delayed;
    buf[writeIdx] = in + g * delayed;
    writeIdx = (writeIdx + 1) % size;
    return output;
  }
};

Choosing delay lengths

The delay lengths for both comb and allpass stages should be mutually prime — no common factors — to avoid modal clustering. For a 48 kHz sample rate, good comb lengths are around 1557, 1617, 1491, 1422, 1277, and 1356 samples. These correspond to roughly 30–35 ms, which gives the natural pre-diffusion ring of a plate reverberator.

For allpass stages, shorter delays work better: 225, 341, and 441 samples (roughly 5–9 ms). Three stages gives good diffusion without excessive computation.

The pre-delay challenge

Plate reverbs have a short pre-delay between the dry signal and the reverb tail — typically 5–30 ms. On the Daisy this is just a circular buffer. The tricky part is smoothing parameter changes: if you change the pre-delay time while audio is playing, you get a pitch shift artifact as the read pointer jumps. The solution is to interpolate the read pointer, or better: crossfade between old and new delay lengths over about 50 ms. 

// Crossfade during pre-delay change
void setPreDelay(float ms) {
  int targetSamples = (int)(ms * SAMPLE_RATE / 1000.f);
  if (abs(targetSamples - currentSamples) > 2) {
    crossfadeTarget = targetSamples;
    crossfadeProgress = 0.f;
  }
}

Putting it all together

The full signal path is:

  1. Input → Pre-delay buffer
  2. Pre-delayed signal → 6 parallel comb filters (summed)
  3. Comb sum → 3 series allpass filters
  4. Allpass output → Stereo spread (L/R with slight time offset)
  5. Wet/dry mix with input

The total compute budget on the Daisy at 48 kHz / 48-sample block size is around 1 ms. This algorithm runs in about 0.3 ms, leaving plenty of headroom for chorus or other effects on the same hardware.

Final notes

For a plate reverb specifically, the character lives in the diffusion density. Increase g on the allpass stages toward 0.7 and you get a smoother, denser tail. Decrease the comb lengths slightly for a brighter, more present reverb. I've been using this module with a sustainer pickup and a long pre-delay (around 60 ms) for a very Gilmour-ish shimmer that sounds nothing like a digital unit.
